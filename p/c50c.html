MySQL数据库七【存储过程】

存储过程

存储过程（Stored Procedure）是一组预编译并存储在数据库中的 SQL 语句集合，可包含数据查询、数据更新、流程控制等操作。它如同数据库中的 “函数”，接受输入参数，执行内部逻辑，并可返回结果。与普通 SQL 语句相比，存储过程具有更高的复用性、安全性和执行效率。

存储过程的作用

提高代码复用性：将常用的业务逻辑封装成存储过程，不同的应用程序或 SQL 语句可重复调用，避免重复编写相同的 SQL 代码。例如，在一个电商系统中，计算订单总金额、统计商品库存等操作可能会在多个地方用到，将这些逻辑封装在存储过程中，就可以方便地在不同模块调用。
增强安全性：应用程序只需调用存储过程，无需直接操作数据库表，减少了敏感数据暴露的风险。同时，可通过权限控制，限制用户对存储过程的调用权限。比如，只允许特定用户执行某个涉及敏感数据操作的存储过程，而不给予其对底层表的直接访问权限。
提升执行效率：存储过程在数据库服务器端执行，减少了客户端与服务器之间的数据传输量。并且，存储过程在首次执行时会被编译并缓存，后续调用可直接执行缓存的代码，加快执行速度。尤其对于复杂的查询或批量数据处理任务，这种优势更为明显。
实现复杂业务逻辑：利用存储过程支持的流程控制语句（如条件判断、循环等），可以实现复杂的业务逻辑，如数据校验、批量处理、事务控制等。以银行转账为例，在一个存储过程中，可以通过事务确保转账操作的原子性，即要么转账成功，要么回滚，保证数据的一致性。

与函数的区别

存储过程和函数在很多方面有相似之处，但也存在一些关键区别：
返回值：存储过程可以通过参数返回多个值，也可以不返回值；函数必须有返回值，且只能返回一个值。例如，一个存储过程可以同时返回用户的姓名、年龄和邮箱等多个信息，而函数一般只能返回一个计算结果，如计算两个数的和。
调用方式：存储过程使用 CALL 语句调用；函数可在 SQL 表达式中直接调用，类似于其他内置函数。比如在查询语句中，可以直接使用函数对某一列数据进行处理，而调用存储过程则需要专门的 CALL 语句。
应用场景：存储过程更适合处理复杂的业务逻辑和批量操作；函数则常用于计算和数据转换，返回特定的计算结果。在处理订单数据时，可能会用存储过程来处理整个订单流程，包括插入订单信息、更新库存等操作；而在计算商品折扣价格时，可能会使用函数来实现。

存储过程的创建与调用

创建存储过程

使用 CREATE PROCEDURE 语句创建存储过程，基本语法如下：

procedure_name：存储过程的名称，在数据库中必须唯一。命名时应尽量遵循一定的规范，使其能清晰地反映存储过程的功能，比如计算订单总金额的存储过程可以命名为calculate_order_total。

  • IN：输入参数，用于将数据传入存储过程，在存储过程内部只能读取，不能修改。例如，要查询某个用户的信息，用户 ID 就可以作为输入参数传入存储过程。
  • OUT：输出参数，用于从存储过程中返回数据，在存储过程内部只能赋值，不能读取传入的值。比如存储过程计算出的结果，像订单总金额，就可以通过输出参数返回给调用者。
  • INOUT：输入输出参数，既可以传入数据，也可以返回数据，在存储过程内部既可读取也可修改。例如，一个存储过程需要对传入的某个数值进行处理并返回修改后的结果，就可以使用 INOUT 参数。
  • data_type：参数的数据类型，如INT、VARCHAR、DATE等，必须与实际传入或返回的数据类型一致。
  • BEGIN和END：用于界定存储过程体的开始和结束，存储过程的具体逻辑代码写在这两者之间。

存储过程的高级应用

事务处理

在存储过程中可以使用事务，确保一组相关操作要么全部成功提交，要么全部失败回滚，保证数据的一致性。
示例：创建一个存储过程，实现银行转账功能，包含事务控制

在这个存储过程中，首先定义了一个异常处理程序，当发生 SQL 异常时，回滚事务并重新抛出异常。然后开始事务，进行两次账户余额的更新操作，如果这两个操作都成功，就提交事务；如果任何一个操作失败，异常处理程序会捕获异常并回滚事务，确保数据不会出现不一致的情况。

游标使用

游标（Cursor）用于处理存储过程中的结果集，它允许我们逐行处理查询返回的数据。
使用语法：
声明游标：DECLARE cursor_name CURSOR FOR select_statement;，游标声明必须在变量声明之后，并且游标必须始终与 SELECT 语句相关联。
打开游标：OPEN cursor_name;
取值：FETCH cursor_name INTO variable1, variable2, …;，将游标当前指向的行数据读取到指定的变量中。
关闭游标：CLOSE cursor_name;，使用完游标后应及时关闭，释放资源。
